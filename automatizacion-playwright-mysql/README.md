# 🗄️ Portafolio de Automatización E2E y Pruebas de Bases de Datos

Este repositorio contiene un framework profesional de automatización de extremo a extremo (E2E) desarrollado con **Playwright y TypeScript**, integrado directamente con una base de datos **MySQL**. O objetivo principal do projeto é mitigar falhas humanas através da validação automatizada de dados nos bastidores.

---

## 🚀 Estructura del Proyecto

### 01. Pruebas de Interfaz de Usuario (Playwright)
*   **Objetivo:** Simulación de flujos críticos de usuario de extremo a extremo mediante interacciones web dinámicas.
*   **Tecnologías:** Desarrollo robusto utilizando TypeScript, garantizando una alta velocidad de ejecución.
*   **Resultado:** Automatización estable com esperas inteligentes, lista para su integración en flujos de trabajo reales.

### 02. Validación de Bases de Datos (MySQL)
*   **Objetivo:** Verificación de la integridad de los datos en los bastidores mediante consultas directas a la base de datos.
*   **Estrategia:** Conexión segura integrada en el script de prueba para ejecutar consultas SQL y validar estados en tiempo real.
*   **Aserciones:** Confirmación exacta de que los datos mostrados en la interfaz gráfica coinciden con los registros guardados en MySQL.

### 03. Infraestructura de Código (TypeScript & Config)
*   **Arquitectura:** Proyecto estructurado de forma limpia con tipado estático avanzado.
*   **Buenas Prácticas:** Configuración sólida del entorno mediante archivos `tsconfig.json` para una resolución de rutas impecable.
*   **Calidad:** Código de automatización optimizado, libre de errores y diseñado bajo estándares de nivel senior.

---

## 🛠️ Tecnologías y Herramientas

*   **Lenguaje:** TypeScript
*   **Framework E2E:** Playwright
*   **Base de Datos:** MySQL
*   **Driver de Conexión:** `mysql2`

---

## 📈 Beneficios Técnicos del Proyecto

*   **Confiabilidad:** Garantiza que las acciones realizadas en la pantalla impacten correctamente la persistencia de los datos.
*   **Independencia de Datos:** Capacidad de preparar y limpiar la masa de datos mediante comandos SQL antes y después de cada ejecución.
*   **Código Limpio:** Configuración optimizada sin errores ni advertencias de tipado no resueltas.
